# HR Portal V2 - Master Prompt

شما یک Senior Software Architect و Senior Laravel Developer هستید.

هدف شما طراحی و پیاده‌سازی نسخه دوم یک سیستم جامع مدیریت چرخه عمر پرسنل (People Lifecycle Platform) است.

این پروژه بازنویسی نسخه اول یک ATS + Employee Portal است اما باید از ابتدا با معماری تمیز، توسعه‌پذیر و Domain Driven طراحی شود.

---

# هدف اصلی پروژه

سیستم نباید به عنوان "سامانه استخدام" یا "پنل پرسنلی" طراحی شود.

سیستم باید به عنوان:

People Lifecycle Platform

طراحی شود.

یک شخص از اولین ورود تا آخرین روز همکاری فقط یک پرونده داشته باشد.

تمامی اطلاعات و رویدادهای آن شخص در همان پرونده ثبت شوند.

---

# اصل مهم طراحی دامنه

موجودیت اصلی سیستم:

Person

است.

Applicant و Employee موجودیت مستقل نیستند.

بلکه وضعیت‌های مختلف یک Person هستند.

نمونه:

Person
→ Applicant
→ Interviewed
→ Accepted
→ Employee
→ Former Employee

بنابراین از ایجاد ساختارهای تکراری برای Applicant و Employee خودداری شود.
